5-Week Microsoft SOC L1/L2 Hybrid Training Program
Overview

This program is designed to build strong theoretical understanding of the Microsoft enterprise SOC stack. It is structured to create a clear operational flow from architecture to detection, investigation, cloud monitoring, and enterprise incident handling.

By the end of this program, you should confidently understand and explain how Microsoft Sentinel, Defender, Entra ID, Azure logs, ServiceNow, and SOAR integrate in a real enterprise SOC environment.

Week 1: Microsoft Security Architecture and Log Pipeline
Objective

Build a complete mental model of how the Microsoft security ecosystem operates.

Core Learning Areas
Azure Tenant and Log Analytics

Understand the Azure tenant and subscription structure at a high level. Learn how Microsoft Entra ID fits into the identity layer.

Understand that Log Analytics Workspace is the central data storage layer where all logs are collected.

Understand that Microsoft Sentinel runs on top of Log Analytics Workspace and uses those logs for detection and investigation.

You must clearly understand that:

Log Analytics Workspace is the data layer.
Microsoft Sentinel is the detection and analysis layer.

Data Connectors

Understand how logs are ingested.

Log Source → Data Connector → Log Analytics Workspace → Sentinel

Know common connectors:

Azure AD

Microsoft Defender

Office 365

Windows Security Events

Syslog

Understand how different log sources map into different tables such as SigninLogs, SecurityEvent, DeviceProcessEvents, OfficeActivity, and AzureActivity.

Component Roles

Understand the difference between:

Microsoft Sentinel as SIEM
Microsoft Defender for Endpoint as EDR
Microsoft 365 Defender as XDR
Sentinel Playbooks as SOAR

Be able to explain SIEM vs EDR vs SOAR clearly.

Alerts vs Incidents

Alert is generated by a rule.

Incident is one or more alerts grouped together for investigation.

Understand:

Severity levels

Entity mapping (User, IP, Host)

Why grouping alerts reduces noise

How incidents are assigned to analysts

End of Week 1 Outcome

You must be able to explain the full operational flow:

Endpoint
→ Defender
→ Log Analytics
→ Sentinel Analytics Rule
→ Alert
→ Incident
→ Analyst Investigation
→ ServiceNow Ticket
