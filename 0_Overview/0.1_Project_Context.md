---

### SOC Runbooks

Runbooks define **step-by-step investigation procedures** for common security incidents.

Example:

**Brute Force Runbook**

1. Check login attempt patterns
2. Verify source IP reputation
3. Check successful login events
4. Review user activity
5. Determine account compromise

---

### Threat Intelligence Enrichment

Analysts verify indicators using:

* IP reputation sources
* malware analysis platforms
* threat intelligence feeds

Example enrichment process:

```
Indicator
↓
Threat Intelligence Lookup
↓
Context Enrichment
↓
Risk Evaluation
```

---

### Service Level Agreements (SLA)

SOC teams must respond to incidents within **defined time limits**.

Example SLA structure:

| Severity | Response Time |
| -------- | ------------- |
| Critical | 15 minutes    |
| High     | 30 minutes    |
| Medium   | 2 hours       |
| Low      | 24 hours      |

Understanding SLAs is essential for **enterprise SOC operations**.

# Sample Detection Query

Example: **Brute Force Login Detection**

```
SigninLogs
| where ResultType != 0
| summarize FailedAttempts = count() by IPAddress, bin(TimeGenerated,5m)
| where FailedAttempts > 10
```

This query identifies **multiple failed authentication attempts from the same IP within a short time window.**
